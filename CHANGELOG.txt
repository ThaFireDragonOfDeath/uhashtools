µHashtools 0.3.0 (13.04.2025):
+ Selecting the target file by passing the filepath as command line
  argument.
+ Included the debugging symbols (.pdb files) in generated
  distribution archives.
+ Automatic CI builds of release archives on pushing into the master
  branch. The built archives can be are uploaded as artifacts and can
  be downloaded from successful workflow runs (see tab "Actions"
  within the GitHub repository).
* Simplified the packaging of the source code in distribution
  archives. The source code is now located directly within the
  "source_code" directory instead within an additional ZIP archive
  in the same directory.
* Declared licenses according to "REUSE Specification - Version 3.3".
* Improved user documentation, developer documentation and the in
  header function documentation.
- Removed release archive generator script (make_release.ps1).
  Distribution archives are now generated by the "dist" target in the
  makefile. Please refer the release procedure documentation for the
  updated information.
